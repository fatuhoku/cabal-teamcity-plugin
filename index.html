Cabal TeamCity Build Runner

(Relatively) painless continuous integration for Haskell.

What's this plugin all about?

TeamCity is a continuous integration server. Cabal is a build and package manager for Haskell programs and libraries. Even though you can quite easily invoke a build of a Cabalised Haskell program by running cabal-install through the command line runner, common operations such as managing and editing of build parameters or digging through long build logs can be a bit of a pain.

Features

Cabal TeamCity Build Runner Plugin presents a tighter integration of the Cabal build process with TeamCity with these features:

  • Effortless configuration: set up and configure a Cabal runner Build Step from a standard Build Configuration page
  • Properties: the cabal-install tool's location and version are exposed as Agent Parameters, allowing you to discriminate between build agents by the version of cabal-install
  • Hierarchical build log: it is now much easier to dig through heaps of build output, thanks to TeamCity presenting the hierarchical structure as of the build log as a with the Tree View
  • Basic test reporting: see the number of successful and failed QuickCheck, QuickCheck 2 and HUnit test runs at a glance for when your test-suite built with Max Bolingbroke's (@batterseapower) wonderful test-framework (site here)

Finally! Quick and easy Continuous Integration for Haskell!

Download and Install

Download latest release

Grab cabalRunner.zip from above or the GitHub repository's Downloads section, and then drop it into either ~teamcity/.BuildServer/plugins directory (UNIX) (where ~teamcity is the home directory of the UNIX user used to run TeamCity Server) or the <TeamCity Data Directory>/plugins (Windows).

Upgrade

Simply overwrite the existing cabalRunner.zip file with the latest released zip file if you've already installed an earlier version of this plugin before.

TeamCity Server sees the change, and will automatically trigger an Agent-side upgrade for the plugin. TeamCity Server however will not upgrade the plugin unless you restart it, so be sure to do that to ensure new version of the plugin becomes activated.
